Sable Command-and-Control

Sable is an authorized-use command-and-control tool written in Go. I used it as a self-study project: an AI-assisted build I ran in an isolated lab to understand how agent/server C2 communication and its security controls actually work. The server receives encrypted agent beacons over HTTPS with DNS as a fallback transport, and exposes operator workflows through a browser-based Web UI and an interactive CLI.

Goal

Understand C2 internals hands-on, in a safe lab setting: how agents beacon, how operators task them, and what security controls offensive tooling needs.

Architecture

Go server, encrypted agents, HTTPS transport, DNS fallback, browser UI, interactive CLI, and a loopback-only operator API boundary.

Security Controls

AES-256-GCM encrypted beacons, HMAC-SHA256 authentication, HKDF-derived keys, nonce caching, timestamp validation, TLS certificate fingerprint pinning, and JWT-protected REST endpoints.

What I Took Away

A working mental model of C2 comms: why you'd layer HTTPS with a DNS fallback, what AES-GCM and HMAC actually buy you, and why replay protection and certificate pinning matter. I can explain the controls and the design decisions. The Go itself was AI-generated, and I don't claim to have hand-written it.

Red Team Runbook

The Red Team Runbook is a living technical notebook I built from lab repetition, HackTheBox practice, and hands-on exploitation work. It's structured for fast recall, without hiding the reasoning behind each technique.

Purpose

Turn my practice notes into a repeatable reference for enumeration, exploitation, privilege escalation, and post-exploitation decisions.

Coverage

Recon and service enumeration, web exploitation, Active Directory attack paths, Linux and Windows privilege escalation, lateral movement, and tunneling.

How I Use It

Commands, prerequisites, indicators, and follow-on checks are grouped by workflow, so it works as a real field reference while I'm testing.

Why I Built It

Writing techniques down in my own words is how I actually learn them. Keeping it public keeps me honest about explaining the reasoning, not just the commands.

Highlighted Areas

  • Web testing notes for XSS, SSRF, SQL injection, file upload, and auth issues.
  • Active Directory workflows for enumeration, Kerberoasting, delegation abuse, and credential attacks.
  • Linux and Windows privilege escalation checks organized around repeatable triage paths.
