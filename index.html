Sable C2 Tool

Sable is an authorized-use command-and-control tool written in Go. The project implements a Sable server that receives encrypted agent beacons over HTTPS with DNS as a fallback transport, then exposes operator workflows through a browser-based Web UI and an interactive CLI.

Goal

Build a lab-safe C2 framework to better understand agent/server communication, operator tasking, and the security controls needed around offensive tooling.

Architecture

Go server, encrypted agents, HTTPS transport, DNS fallback, browser UI, interactive CLI, and a loopback-only operator API boundary.

Security Controls

AES-256-GCM encrypted beacons, HMAC-SHA256 authentication, HKDF-derived keys, nonce caching, timestamp validation, TLS certificate fingerprint pinning, and JWT-protected REST endpoints.

What It Shows

Secure transport design, key-handling decisions, API boundary thinking, and the ability to build usable operator workflows around low-level security mechanics.

Next Improvements

  • Role-based operator access for separating administrative and tasking actions.
  • Improved task queue observability for session state, errors, and response timing.
  • Broader integration coverage around agent lifecycle and transport failover behavior.

Red Team Reference Manual

The Red Team Reference Manual is a living technical notebook built from lab repetition, HackTheBox practice, and hands-on exploitation workflows. It is structured to support fast recall without hiding the reasoning behind each technique.

Purpose

Convert practice notes into a repeatable reference for enumeration, exploitation, privilege escalation, and post-exploitation decision making.

Coverage

Web exploitation, network enumeration, Active Directory attack paths, Linux privilege escalation, and Windows privilege escalation.

UX Focus

Commands, prerequisites, indicators, and follow-on checks are grouped by workflow so the manual works as a practical field reference.

Portfolio Value

Shows consistent hands-on practice, organized technical writing, and the ability to translate exploitation steps into reusable guidance.

Highlighted Areas

  • Web vulnerability testing notes for XSS, SSRF, SQL injection, file upload, and auth issues.
  • Active Directory workflows for enumeration, Kerberoasting, delegation abuse, and credential attacks.
  • Linux and Windows privilege escalation checks organized around repeatable triage paths.
